The hardest feature in my staff rostering system wasn't the frontend or the API — it was the automated scheduler. I needed to assign employees to shifts while respecting availability windows, gender requirements, rest periods, maximum hours, and balanced workload distribution. All at once.

Trying to solve this with nested loops and if-statements was a dead end. Then I discovered constraint programming.

## The Problem

Imagine you have 20 employees and 50 shifts in a week. Each employee has different availability. Some wards require specific genders. Nobody should work back-to-back shifts without 8 hours rest. And you want to maximise total coverage while keeping workloads balanced.

The number of possible assignments is astronomical. Brute-forcing it would take longer than the heat death of the universe. You need an algorithm that's smarter than trying every combination.

## Enter Google OR-Tools

Google OR-Tools is an open-source optimisation library. Its CP-SAT solver (Constraint Programming with Satisfiability) lets you define variables, constraints, and an objective — then it finds the best solution automatically.

The mental model is simple:

1. **Variables**: For each employee-shift pair, create a boolean variable — "does this employee work this shift?"
2. **Constraints**: Add rules that must be true (availability, rest periods, one employee per shift, etc.)
3. **Objective**: Maximise the number of shifts that get covered

```python
from ortools.sat.python import cp_model

model = cp_model.CpModel()

# Create a variable for each possible assignment
assignments = {}
for emp in employees:
    for shift in shifts:
        assignments[(emp.id, shift.id)] = model.NewBoolVar(
            f'emp_{emp.id}_shift_{shift.id}'
        )
```

## Defining Constraints

Each constraint translates to a simple rule in the model:

**One employee per shift:**
```python
for shift in shifts:
    model.AddAtMost(
        [assignments[(emp.id, shift.id)] for emp in employees],
        1
    )
```

**Only assign if available:**
```python
if not is_available(emp, shift):
    model.Add(assignments[(emp.id, shift.id)] == 0)
```

**Minimum 8-hour rest between shifts:**
This was the trickiest one. For each pair of overlapping or too-close shifts, I added a constraint that the same employee can't be assigned to both.

## The Result

The solver runs in under 2 seconds for a typical week's schedule with 20+ employees and 50+ shifts. It finds an optimal (or near-optimal) assignment that respects every constraint.

What would have taken me weeks to code with hand-written logic took a few hours with OR-Tools. The solver is also provably correct — it won't violate constraints, ever. My hand-written logic definitely would have had edge-case bugs.

## What I Learned

Constraint programming is one of those tools that feels like magic the first time you use it. The key insight is that **you describe what you want, not how to get it**. You tell the solver your rules and it figures out the rest.

If you have any kind of scheduling, assignment, or resource allocation problem, look into OR-Tools before writing custom logic. The learning curve is a day or two. The time saved is enormous.
