What I Learned Building My First FastAPI Backend

From 'what is a route?' to JWT auth and constraint solvers — here's everything I picked up building a full production backend for the first time.

Six months ago I'd never built a backend from scratch. Today I have a full FastAPI application with JWT authentication, role-based access control, a PostgreSQL database, and an AI-powered scheduling engine. Here's what the journey actually looked like.

Why FastAPI?

I considered Flask and Django, but FastAPI won me over for three reasons: automatic Swagger docs at /docs, built-in request validation with Pydantic, and async support out of the box. For a first backend project, having auto-generated API documentation was incredibly helpful — I could test endpoints without writing a single line of frontend code.

The Things That Tripped Me Up

Database sessions and dependency injection

FastAPI's dependency injection system is elegant once you understand it, but it took me a while to grasp why I needed a get_db generator function:

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

The yield keyword confused me at first. Why not just return the session? The answer is cleanup — yield lets FastAPI close the session after the request finishes, even if something goes wrong. This pattern prevents database connection leaks.

JWT authentication

Implementing JWT auth from scratch taught me more about authentication than any tutorial. The flow is straightforward in theory — user logs in, gets a token, sends it with every request — but the details matter. Token expiry, password hashing with bcrypt, extracting the current user from the token header... each piece has gotchas.

My biggest mistake? Storing the plain password in the database during early development "just to test things." Don't do this. Set up proper hashing from day one.

SQLAlchemy relationships

Defining relationships between models (employees, shifts, assignments, availability) was the hardest conceptual leap. Understanding the difference between relationship() and ForeignKey, and when to use back_populates, took several evenings of reading docs and experimenting.

What I'd Do Differently

If I started again, I'd set up database migrations with Alembic from the very beginning. I ended up manually recreating tables several times when I changed schemas, which was painful and error-prone.

I'd also write tests earlier. I left testing until late in the project and regretted it — having even basic endpoint tests would have caught bugs much faster.

The Takeaway

FastAPI is genuinely excellent for learning backend development. The auto-generated docs, type safety, and clear error messages make the feedback loop tight. If you're a frontend developer thinking about learning backend, I'd strongly recommend starting here.
